Introducción a POI

La misión del proyecto Apache POI es crear y mantener la API Java para la manipulación de varios formatos de archivo basado en los estándares abiertos de Office XML (OOXML) de Microsoft OLE y el formato de Documento 2 compuesto (OLE2). En resumen, usted puede leer y escribir archivos MS Excel utilizando Java. Además, usted puede leer y escribir MS Word y MS PowerPoint usando Java. Tenemos una completa API para portar otros formatos de OOXML OLE2 y otros invitados a participar.

OLE2 archivos incluyen la mayoría de archivos de Microsoft Office tales como XLS, DOC, PPT y así como la API de serialización MFC formatos de archivo basados. El proyecto provee un API para el sistema de ficheros OLE2 (POIFS) y OLE2 Propiedades del documento (HPSF).

Esto incluye XLSX, DOCX y PPTX. The project provides a low level API to support the Open Packaging Conventions using openxml4j . El proyecto proporciona un bajo nivel de la API para apoyar a los Convenios de Open Packaging utilizando openxml4j.

Para cada aplicación MS Office, existe un módulo de componentes que trata de proporcionar un elevado nivel común de la API Java para ambas OLE2 y formatos de documentos OOXML. Esta es la más desarrollada para los libros de Excel (SS = HSSF + XSSF). Avanzan los trabajos para los documentos de Word (HWPF + XWPF) y presentaciones de PowerPoint (HSLF + XSLF).

El proyecto recientemente ha añadido soporte para Outlook (HSMF). Microsoft abre las especificaciones de este formato en octubre de 2007.

También hay proyectos para Visio (HDGF) y Editor (HPBF).
