Message-driven Beans

Los MDB también se usan para la lógica de negocio, pero a diferencia de los session beans nunca son llamados directamente por el cliente, sino que se crean como resultado de ciertos mensajes enviados al servidor.

Permiten el envío de mensajes asíncronos entre sistemas, como por ejemplo indicar al sistema de un cliente que hemos enviado la mercancía. Ejemplo:

@MessageDriven(mappedName = "MDBAuditoria", activationConfig = {
@ActivationConfigProperty(propertyName = "acknowledgeMode", propertyValue = "Auto-acknowledge"),
@ActivationConfigProperty(propertyName = "destinationType", propertyValue = "javax.jms.Queue")
})
@Interceptors(ej4.AuditoriaListener.class)
public class MDBAuditoriaBean implements MessageListener {
public MDBAuditoriaBean() {
}
public void onMessage(Message message) {
try {
TextMessage text = (TextMessage) message;
String msg = text.getText();
System.out.println("MENSAJE="+msg);
} catch (Exception ex) {
ex.printStackTrace();
}
}
}
