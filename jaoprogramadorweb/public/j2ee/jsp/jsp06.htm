Directivas

Hemos sido plenamente calificar la java.util.Date en los ejemplos en las secciones anteriores. Tal vez usted se pregunta por qué no nos limitamos a importar java.util.*;

Es posible utilizar "importar" las declaraciones en JSP, pero la sintaxis es un poco diferente de Java normal. Pruebe el ejemplo siguiente:

<%@ page import="java.util.*" %>
<HTML>
<BODY>
<%
System.out.println( "Evaluating date now" );
Date date = new Date();
%>
Hola hoy es: <%= date %>
</BODY>
</HTML>

La primera línea en el ejemplo anterior se llama una "Directiva". Una "directiva JSP" comienza con <% @ caracteres.

Esta es una "directiva de página". La directiva de página puede contener la lista de todos los paquetes importados. Para importar más de un artículo, los nombres de paquetes separados por comas, por ejemplo,

<%@ page import="java.util.*,java.text.*" %>

Hay una serie de directivas JSP, además de la directiva de página. Además de las directivas de pÃ¡gina, las directivas de otros más útiles son incluir y taglib. Estaremos cubriendo taglib por separado.

La directiva include se utiliza para físicamente incluir el contenido de otro archivo. El archivo incluido puede ser HTML o JSP o cualquier otra cosa, el resultado es como si el fichero JSP original que realmente contienen el texto incluido. Para ver esta directiva en la acción, crear un JSP nuevo

<HTML>
<BODY>
<BR> Va a incluir hello.jsp ... <BR>
<% @ page import = "hello.jsp"%>

</BODY>
</HTML>

Ver este JSP en tu navegador y podrás ver tu hello.jsp originales se incluyen en el JSP.
