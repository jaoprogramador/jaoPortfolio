La protección de su sitio web con una página de acceso

Algunos sitios requieren que todos los usuarios registrarse con un nombre de usuario y contraseña, antes de poder visitar cualquier página.

Esto puede hacerse a través de sesiones JSP o servlets, y de hecho se trataba de una técnica común para un tiempo. Pero a partir de una nueva versión de las especificaciones de Servlets (2,2) de Sun, esta característica es ahora muy sencillo de implementar.

Ya no es necesario utilizar técnicas de JSP a proporcionar nombre de usuario / contraseña de protección, pero todavía es un requisito muy común de sitios web, por lo tanto una breve descripción se proporciona aquí.

Para proteger mediante contraseña de su sitio, sólo tiene que diseñar una página de acceso. Esta página puede ser tan simple o complicada que necesite que sea. Debe contener una etiqueta <FORM>, con el método establecido para publicar y las actuaciones previstas a "j_security_check".

<FORM METHOD=POST ACTION=j_security_check>

El j_security_check objetivo es proporcionada por el servidor de aplicaciones, y no deben ser codificadas.

El formulario debe contener dos campos <INPUT>, llamado j_username y j_password, respectivamente, para el nombre de usuario y contraseña. Normalmente, el campo nombre de usuario será un campo de texto, y el campo de la contraseña será un campo de contraseñas.

Después de esto, usted debe decirle a su servidor de aplicaciones para proteger con contraseña sus páginas mediante la página de acceso que nos ha facilitado. Los detalles varían de un servidor a otro, pero una buena aplicación le proporcionará los ganchos que se pueden utilizar, por ejemplo, para hacer coincidir los nombres de usuario y contraseñas en contra de una base de datos. (Por ejemplo, en Blazix usted puede proporcionar una implementación de la interfaz de desisoft.deploy.AuthCheck para comprobar los nombres de usuario y contraseñas en contra de una base de datos u otras fuentes.)
