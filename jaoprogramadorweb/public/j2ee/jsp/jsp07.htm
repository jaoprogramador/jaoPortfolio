Declaraciones JSP

La JSP escribir se convierte en una definición de clase. Todos los scriptlets escribir se colocan dentro de un único método de esta clase.

También puede agregar variables y declaraciones de métodos de esta clase. A continuación, puede utilizar estas variables y los métodos de su scriptlets y expresiones.

Para añadir una declaración, debe utilizar el <%! Y%> secuencias de incluir su declaración, como se muestra a continuación.

<%@ page import="java.util.*" %>
<HTML>
<BODY>
<%
getDate Fecha () (System.out.println ( "En getDate () método" ); regreso theDate;)
%> Hola! The time is now <% = getdate ()%>
</BODY>
</HTML>
El ejemplo se ha creado un poco forzado, para mostrar variables y declaraciones de métodos.

Aquí estamos declarando un theDate Fecha variable, y el getDate método. Ambas están disponibles ahora en nuestro scriptlets y expresiones.

Pero este ejemplo ya no funciona! La fecha será la misma, no importa cuantas veces volver a cargar la página. Esto es así porque se trata de declaraciones, y sólo se evaluarán una vez cuando se carga la página. (Como si estuviera creando un de clase y se habrá declarado la inicialización de variables en ella.)

Nota: Ahora que usted sabe cómo hacer esto, en general, no es una buena idea utilizar las variables como se muestra aquí. La JSP normalmente se ejecutaría con múltiples subprocesos de una sola instancia. Diferentes hilos que interferir con el acceso variable, porque va a ser la misma variable para todos ellos. Si usted tiene que usar variables en JSP, debe utilizar sincronizados de acceso, pero que perjudica el rendimiento. En general, los datos que necesite debe ir bien en el periodo de sesiones o el objeto de la solicitud (se introducen estas un poco más tarde) si pasa datos entre diferentes páginas JSP. Variables que usted declare scriptlets interior están muy bien, por ejemplo <% int i = 45;%> porque estos se declaran dentro del ámbito local y no son compartidos.
