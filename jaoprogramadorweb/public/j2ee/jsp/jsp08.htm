Etiquetas JSP

Otro elemento importante de la sintaxis son las etiquetas JSP. Etiquetas JSP no use <%, pero sólo el carácter <. Una etiqueta de JSP es algo así­ como una etiqueta HTML. Etiquetas JSP puede tener una "etiqueta de inicio", un "organismo de etiqueta" y una "etiqueta de cierre". La etiqueta de inicio y final tanto el uso del nombre de etiqueta, dentro de caracteres <y>. El final comienza con el carácter de un / después de que el carácter <. Los nombres de las etiquetas tienen un carácter integrado de colon: en ellos, la parte antes de los dos puntos se describe el tipo de la etiqueta. Por ejemplo:

<some:tag>
cuerpo
</some:tag>

Si la etiqueta no necesita un cuerpo, el inicio y el final puede ser convenientemente fusionado, como

<some:tag/>

A continuación el cierre de la etiqueta de apertura con /> en lugar de carácter>, estamos terminando la etiqueta de inmediato, y sin cuerpo. (Esta convención es la sintaxis de la misma como XML.)

Etiquetas pueden ser de dos tipos: carga de una biblioteca de etiquetas externas, o etiquetas predefinidas. Etiquetas predefinidas empezar con jsp: caracteres. Por ejemplo, jsp: include es una etiqueta predefinida que se utiliza para incluir otras páginas.

Ya hemos visto la directiva include. Jsp: include es similar. Pero en lugar de cargar el texto del archivo incluido en el archivo original, se llama al objetivo que figuran en tiempo de ejecución (la forma de un navegador llamará el objetivo incluido. En la práctica, esto es en realidad una petición de simulación en lugar de una ronda completa-ida y vuelta entre el navegador y el servidor). El siguiente es un ejemplo de jsp: incluyen el uso de

<HTML>
<BODY>
<BR> Va a incluir hello.jsp ... <BR>
<jsp:include page="hello.jsp"/>
</BODY>
</HTML>

Pruebelo y vea lo que hay. Ahora cambie la "jsp: include" a "jsp: forward" y ver cual es la diferencia. Estas dos etiquetas predefinidas son con frecuencia muy útil.
