Session

En un sitio Web típico, un usuario puede visitar varias páginas y realizar varias interacciones.

Si está programando el sitio, es muy útil para poder asociarse con algunos datos de cada visitante. Para ello, la "session" s puede ser utilizado en JSP.

Una sesión es un objeto relacionado con un visitante. Los datos pueden ser introducidos en la sesión y se recuperan de ella, como un Hashtable. Un conjunto diferente de los datos se mantienen para todos los visitantes del sitio.

AquÃ­ hay un conjunto de páginas que poner el nombre de un usuario en la sesión, y mostrarlo en otros lugares. Pruebe la instalación y uso de estos.

En primer lugar tenemos un formulario, llamémosle GetName.html

<HTML>
<BODY>
<FORM METHOD=POST ACTION="SaveName.jsp">
¿Cual es tu nombre? <INPUT TYPE=TEXT NAME=username SIZE=20>
<P><INPUT TYPE=SUBMIT>
</FORM>
</BODY>
</HTML>

El objetivo de la forma es "SaveName.jsp", que guarda el nombre del usuario en la sesión. Tenga en cuenta la variable de sesión. Esta es otra variable que se hace normalmente disponibles en las páginas JSP, al igual que las variables especiales. (En el directiva @ Page, que puede indicar que usted no necesita periodos de sesiones, en la que el "periodo de caso" variable no estará disponible.)

<%
String name = request.getParameter ( "username");
session.setAttribute( "theName", name );
%>
<HTML>
<BODY>
<A HREF="NextPage.jsp"> Continuar </ A>
</BODY>
</HTML>

El SaveName.jsp guarda el nombre del usuario en la sesión, y pone un enlace a otra página, NextPage.jsp.

NextPage.jsp muestra como recuperar el nombre guardado.

<HTML>
<BODY>
Hello, <%= session.getAttribute( "theName" ) %>
</BODY>
</HTML>

Si usted trae dos navegadores diferentes (no diferentes de un mismo navegador), o ejecutar dos navegadores de dos máquinas diferentes, usted puede poner un nombre en un navegador y otro nombre en otro navegador, y ambos nombres se mantendrá la pista.

La sesión se mantiene en torno a hasta un periodo de tiempo de espera. Luego, se supone que el usuario ya no es visitando el sitio, y la sesión se descarta.
