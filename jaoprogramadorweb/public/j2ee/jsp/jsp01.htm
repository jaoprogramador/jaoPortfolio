Familiarizarse con el servidor de JSP

Si usted no tiene una red capaz de JSP-servidor o servidor de aplicaciones, el primer paso es descargar uno. Hay muchos servidores disponibles, por ejemplo, la mayoría de los cuales se puede descargar gratuita de evaluación y / o de desarrollo. Algunas de ellas son:


- Blazix de Desiderata Software (1,5 megabytes, JSP, Servlets y EJB).
- TomCat from Apache ( Approx 6 Megabytes ) TomCat de Apache (aprox 6 megabytes).
- WebLogic de BEA Systems (Aprox 40 Megabytes, JSP, Servlets y EJB.
- De WebSphere de IBM (aprox. 100 Megabytes, JSP, Servlets y EJB).


Para sacar el mejor provecho de este tutorial, se recomienda que todo el material debe ser tratado con un servidor real.

Una vez que haya una red capaz JSP-servidor o servidor de aplicaciones, lo que necesita saber la siguiente información al respecto:
- Dónde colocar los archivos
- Cómo acceder a los archivos desde el navegador (con un http:prefix, no como archivo).

Usted debe ser capaz de crear un simple archivo, tales como:

<HTML>
<BODY>
Hola JAO
</BODY>
</HTML>

saber dónde colocar este archivo y la forma de ver en el navegador con un prefijo http://.

Dado que este paso es diferente para cada servidor web, usted tendrá que ver la documentación del servidor web para averiguar cómo se hace esto.
