Formularios

Introducción

Como sabemos, para crear un formulario en html, debemos utilizar <FORM ... En php se utiliza el mismo formato pero indicando siempre su envío mediante la función print (); Veamos un formulario sencillo en html:

<HTML>
<HEAD>
<TITLE>Formulario HTML </TITLE>

</HEAD>
<BODY>
<FORM ACTION="recibir.php" METHOD=POST>
Nombre: <INPUT TYPE=TEXT NAME="Nombre" SIZE=20><BR>
Apellido: <INPUT TYPE=TEXT NAME="Apellido" SIZE=40><BR>
Emilio: <INPUT TYPE=TEXT NAME="Email" SIZE=60><BR>
Comentarios: <TEXTAREA NAME="Chascarrillos" ROWS=5 COLS=40></TEXTAREA><BR>
<INPUT TYPE=SUBMIT NAME="SUBMIT" VALUE="Enviar">
</FORM>
</BODY>
</HTML>

Comenzamos con las etiquetas habituales y en la línea 6 definimos el formulario. Al hacerlo seleccionamos su destino: ACTION="recibir.php" y el método de envío: METHOD=POST Desde html puro estamos acostumbrados a enviar correo tecleando ACTION =”mailto:nombre@dominio.xx” aquí lo enviaremos a una página php que crearemos posteriormente: recibir.php.

En cuanto al método de envío, disponemos de GET y POST. Las diferencias fundamentales son que con GET el volumen de datos está bastante limitado y, además, envía los datos del formulario junto con la dirección web a la que van dirigidos, mientras que POST lo hace de forma invisible para el usuario. Esto quiere decir que desde el punto de vista de la seguridad es más recomendable POST.

Podríamos haber codificado la página en php:

<?php
print ("<HTML><HEAD><TITLE>Formulario HTML</TITLE></HEAD><BODY>\n");
print ("<FORM ACTION=\"recibir.php\" METHOD=POST>\n");
print ("Nombre <INPUT TYPE=TEXT NAME=\"Nombre\" SIZE=20><BR>\n");
print ("Apellido <INPUT TYPE=TEXT NAME=\"Apellido\"
SIZE=40><BR>\n");
print ("Emilio <INPUT TYPE=TEXT NAME=\"Email\" SIZE=60><BR>\n");
print ("Comentarios <TEXTAREA NAME=\"Chascarrillos\"
ROWS=5 --> COLS=40></TEXTAREA><BR>\n");
print ("<INPUT TYPE=SUBMIT NAME=\"SUBMIT\" VALUE=\"Enviar\">\n");
print ("</FORM>");
print ("</BODY></HTML>");
?>

Fíjate que en realidad utilizamos código html 100%, únicamente añadimos los print necesarios para que el servidor envíe el código html necesario al navegador. Fíjate en los \” y \n.

Veamos recibir.php

<HTML>
<HEAD>
<TITLE>Resultados del formulario</TITLE>
</HEAD>
<BODY>
<?php
/* Esta página recibe los datos generados por enviar.php o enviar.html. */
print ("Tu nombre es $Nombre.<BR>\n");
print ("Te apellidas $Apellido.<BR>\n");
print ("Tu dirección de correo electrónico es $Email.<BR>\n");
print ("Esto son los chascarrillos que has contado:<BR>\n $Chascarrillos<BR>\n");
?>
</BODY>
</HTML>

En primer lugar, como en el caso anterior, podríamos haber creado las etiquetas de inicio y fin de página desde php con sendos print(); A efectos prácticos el resultado va a ser el mismo. En este script hacemos que aparezcan en pantalla los valores de las variables $Nombre, $Apellido, $Email y

$Chascarillos. Estos valores los obtiene de los campos creados con el mismo nombre en enviar.php/html

GET sin formulario

También podemos enviar valores sin necesidad de utilizar un script. Para ello bastaría con añadir a la URL el nombre de la variable que vamos a utilizar con su correspondiente valor. Veamos un ejemplo:

<HTML>
<HEAD>
<TITLE>Resultados del formulario</TITLE>
</HEAD>
<BODY>
<?php
/* Entrada de datos desde la URL mediante GET */
print ("hola $nombre");
?>
</BODY>
</HTML>
En el navegador podemos ver que creamos la variable $nombre simplemente tecleando al final de la URL ?nombre=Carlos que php nos mostrará en pantalla. Si no hicieramos esto únicamente mostraría hola

También podemos enlazar una página con otra enviandole valores para las variables que tengamos predefinidas en ella: <a href=”saludo.php?nombre=Carlos”>Clic para saludo<a>

El número de variables es indeterminado. Si queremos enviar varias podemos hacerlo utilizando el símbolo & La única precaución a tomar es que si hubiera espacios en blanco deberíamos reemplazarlos por el signo + http://localhost/php/get.php?nombre=Carlos&apellido=Muñoz
