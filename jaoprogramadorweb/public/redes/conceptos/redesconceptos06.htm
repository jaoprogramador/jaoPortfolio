Implantación de redes de área local

1.-¿Qué es un sistema operativo de red NOS?
Es el software que hace que un sistema informático pueda comunicarse con otros equipos en el ámbito de una red.
2.-¿Qué es un programa de comunicaciones?¿Ejemplos?
Es un programa en el que los usuarios pueden hacer transferencia de ficheros a través de líneas serie sin necesidad de adquirir productos añadidos.
Interlink-Interserver (DOS)
3.-Evolución de los sistemas operativos NOS.
ETAPA INICIAL: Se necesitaba software especial para enviar grandes volúmenes de datos.
INTEGRACIÓN DE LOS PROGRAMAS DE COMUNICACIONES EN EL DOS: Se utilizan programas de comunicaciones.
SOFTWARE DE RED AÑADIDO: Se requieren productos de red que deben instalarse sobre el DOS o sobre Windows.
ACTUALMENTE: Los sistemas operativos de red están perfectamente integrados.
4.-¿Qué es un grupo de trabajo en un entorno de red?
Es una agrupación de recursos con objeto de facilitar la gestión de los mismos.
5.-¿Cuál es el concepto de dominio en una red?
Es una extensión del concepto de grupo de trabajo
Centralización de la gestión de la red
Facilidad para la administración de los equipos
Control de usuarios y contraseñas
Jerarquización de los recursos
6.-¿Qué es un servicio de directorio?
Es una base de datos jerárquica y organizada en forma de objetos que contienen información fácilmente accesible y útil para el sistema operativo, la red o los usuarios que solicitan los recursos del sistema.
7.-Enumera los elementos de hardware necesarios para instalar una red.
Adaptadores de red, las estaciones de trabajo, los servidores de red y los clientes ligeros.


8.-¿Qué es un adaptador de red, tarjeta de red o NIC? ¿Dónde se coloca?
Es un interface hardware, entre la plataforma o sistema informático y el medio de transmisión físico, por el que se transporta la información de un lugar a otro.

Se inserta en el bus de comunicaciones del orednador personal (ranuras de expansión), aunque en ocasiones el propio equipo posee adaptadores de red.
9.-Enumera las partes de una tarjeta de red.
- Interface de conexión al bus del ordenador (Ranuras de expansión)
- Interface de conexión al medio de transmisión (Conectores)
- Componentes electrónicos internos, propios de la trajeta
- Elementos de configuración de la tarjeta: Puentes, conmutadores...
10.-¿Qué es un slot de conexión?
Es el conector físico en donde se inserta la tarjeta.
El slot de conexión debe coincidir con el interface de la tarjeta.
11.-¿Cuál es el slot de conexión que más se usa actualmente?¿Cuáles son sus características?
El slot es el PCI.
Bus de 32 bits, 40 Mhz y 120 Mbps. Actualmente han aparecido estándares PCI de 64 Mbps.
12.-¿Qué necesita una tarjeta de red para comunicarse con el sistema operativo del sistema en el que se inserta?
Un driver o controlador.
13.-Enumera alguna de las funciones que realiza una tarjeta de red.
- Procesa la información que le llega procedente del bus del ordenador, para producir una señalización adecuada en el medio de transmisión.
- Realiza la confección de paquetes
14.-¿Cuáles son los principales parámetros que deben ser configurados en una tarjeta de red?
- IRQ: Interrupciones
- Dirección de E/S (entrada/salida): Intercambio de datos
- DMA: Acceso directo a memoria

- Tipo de transceptor
15.-¿Qué es una interrupción IRQ?
Es el número de una línea de interrupción con el que se avisan sistema y tarjeta de que se producirá un evento de comunicación entre ellos.
16.-Qué es una dirección de entrada/salida E/S? ¿En qué se suele expresar?
Es una dirección de memoria en la que se escriben y leen tanto el procesador central del sistema como la tarjeta, de modo que les sirve de bloque de memoria para el intercambio mutuo de datos.
Se suele expresar en HEXADECIMAL.

Se configuraban a través de puentes (jumpers) y conmutadores (switches).
Actualmente la configuración es por software.

24.-¿Qué es un servidor de red?
Es un nodo especializado en brindar servicios al resto de los nodos de la red.
25.-¿Cuáles son los servicios más comunes de un servidor de red?
Discos, ficheros, impresoras y comunicaciones.
26.-¿Qué características se le piden a un servidor de red?
- Potencia de proceso: A veces poseen más de un microprocesador
- Memoria RAM: 64 Mbytes como mínimo
- Capacidad de almacenamiento en disco
- Conexión a la red: Debe der eficaz puesto que soportará todo el tráfico de la red
27.-¿Qué software se carga sobre las tarjetas de red?
Se carga un software controlador.
28.-¿Qué es un controlador o driver de red?
Es un programa de muy bajo nivel en el que las aplicaciones, mediante el sistema operativo, tienen controlados los accesos al hardware del sistema, y en concreto, a la red.
29.-¿Qué son las pilas o stacks de protocolos?
Son fraccionamientos del software en capas de programas que permiten que sobre la misma tarjeta de red, puedan soportarse distintos protocolos sin interferencias entre ellos.
30.-¿Cuáles son las tecnologías básicas para realizar el enlace entre las capas de alto nivel y la tarjeta?
NDIS (Network Driver Interface Specification) Microsoft y 3COM
ODI (Open Datalink Interface) Novell y Apple
31.-¿Qué es el redirector?
Es el encargado de realizar la transparencia de software entre servicios remotos y locales.

34.-Características del protocolo IP
- Protocolo del nivel de red (Arpanet)
- Protocolo sin conexión por lo que carece de seguridad
- Protocolo preparado para que sus paquetes sean encaminados
35.-Partes de un datagrama IP
Un datagrama IP se divide en 2 partes: Cabecera y campo de datos.
36.-Explicación de los campos de la cabecera de un paquete IP
- Versión: Versión del protocolo. Puede ser Ipv4 o Ipv6
- IHL (Longitud de la cabecera): El valor puede oscilar entre 5x32 bits y 15x32 bits
- Tipo de servicio: Escoge qué tipo de servicio le da más importancia, bien seguridad o bien velocidad.
- Longitud total: Contiene la longitud total del datagrama. Máxima longitud es 64 Kbytes.
- Identificación: Cuando un datgrama se fragmenta, es necesario que todas las partes lleven la misma identificación.
- Bit D/F (bit de no fragmentación): Cuando se pone a 1, indica que el datgrama no ha de ser fragmentado.
- Bit M/F (Bit de más fragmentos): Este bit estará a 1 en todos los fragmentos en que se haya dividido un datgrama, con excepción del último fragmento.
- Desplazamiento del fragmento: Indica el lugar del datagrama en que debe ser insertado un fragmento, para que su ensamblaje final sea correcto.
- Tiempo de vida: Contador que determina la vida de un paquete. Este contador va disminuyendo por cada router que pasa. Con esto se evita que un paquete quede atrapado en la red.
- Protocolo: Indica el protocolo de transporte que ha generado el datagrama (TCP, UDP).
- Código de redundancia cíclica de la cabecera: CRC de la cabecera del paquete IP.
- Dirección fuente: Codifica la dirección de nivel de red del host que produce el paquete.
- Dirección de destino: Codifica la dirección del host destinatario.
- Opción: Campo para posibles ampliaciones.
37.-¿Cómo es una dirección IP?
Cada dirección IP consta de 32 bits agrupados en grupos de 8 bits 81 byte), por lo tanto se expresa con 4 números decimales separados por puntos.
Cada número decimal varía entre 0 y 255, aunque hay algunas restricciones.
Cada dirección IP codifica una red y un host dentro de esa red.
38.-Características de las redes clase A
El primer bit es un 0.
Con los 8 primeros bits se codifica la red y con los 24 restante el host.
39.-Características de las redes clase B
Los dos primeros bits son 10.
Con los 16 primeros bits se codifica la red y con los 16 restantes el host.
40.-Características de las redes clase C
Los tres primeros bits son 110.
Con los 24 primeros bits se codifica la red y con los 8 restantes el host.
41.-¿Por qué solo existen 126 redes de clase A cuando con 7 bits se pueden codificar 128?.

42.-¿Para qué sirve la máscara de subred?
Es una secuencia de 32 bits que sirve para distinguir con facilidad qué parte de una dirección codifica la subred y cuál el host.
43.-¿Qué es el protocolo ICMP? ¿Para qué sirve?
ICMP (Internet Control Message Protocol, protocolo de mensajes de control entre redes): Es un protocolo que encapsula en un único paquete IP algún evento que se produce en la red. Es un protocolo de supervisión.
44.-Características del protocolo TCP
TCP(Transport Control Protocol): Fue diseñado para realizar las conexiones en redes inseguras (Tipo C). Es un protocolo CON conexión.

45.-Características del protocolo UDP
UDP(User Datagram Protocol, protocolo de datagrama de usuario):
- Protocolo SIN conexión que permite la transmisión de mensaes sin garantía de entrega.
- Actúa de interface entre los procesos de los usuarios de la red y el protocolo IP.
46.-¿Qué hace el protocolo ARP?
ARP(Address Resolution Protocol, protocolo de resolución de direcciones): El protocolo envía un paquete IP a todos los nodos de la red, y el nodo que tenga esa IP, contesta con otro paquete ARP de respuesta con su dirección MAC.
El protocolo contrario es el RARP.
47.-¿Para qué sirve el protocolo NFS?
NFS(Network File System, sistema de ficheros de red): Permite servir discos locales al resto de una red, de modo que a los usuarios remotos les parece que son propietarios del disco al que se conectan.
48.-¿Qué parámetros son necesarios para configurar una dirección TCP/IP?
- Dirección IP del host: Suministarda por el administrador de red de la instalación.
- La máscara de subred: Determinara cómo identificar lo que es red de lo que es host.
- La dirección de una puerta de enlace o gateway: Cuando llega un paquete IP con una dirección desconocida, este paquete es retransmitido por esta puerta de enlace que lo enviará a su correspondiente red o lo drenará.
49.-¿Qué hace el protocolo DHCP?
DHCP(Dynamic Host Configuration Protocol, protocolo de configuración dinámica de host): Ayuda al administrador de red para automatizar las asignaciones de direcciones IP, haciéndolas dinámicamente, es decir, no son Ips fijas.
50.-¿Qué hace un servidor DNS?
DNS(Domain Name System, sistema de nombres de dominio): Convierte una dirección IP en lenguaje articulado para que se pueda entender o memorizar mejor.


51.-Indica las partes y el significado de cada una en una dirección DNS.
Una dirección DNS consta de 2 partes:
La primera parte identifica al nodo dentro de la subred.
La segunda parte identifica a la subred y se llama “dominio”.
La proliferación de nodos en la internet ha creado la necesidad de fraccionar los dominios en sub-dominios. Cada uno de los niveles va separado del siguiente nivel por un punto en la escritura del nombre.

52.-¿Qué es un servidor Wins?
Wins(Servicio de nombres de Internet de Windows): Es un sistema de nombres planos, no articulados, que identifican cada nodo de una red por un nombre único.
Es un servicio de Microsoft que utiliza esto para evitar el tráfico de paquetes de difusión.

68.-¿Qué es un dominio Microsoft?
Es un grupo de ordenadores que comparten la misma base de datos de cuentas de usuario, privilegios, derechos de acceso, seguridad, etc.
69.-¿Qué es un servicio de directorio?
Es la gestión de todos los elementos que comparten el dominio.
70.-¿Qué diferencia existe entre un dominio y un grupo de trabajo en Microsoft?
En un dominio el grupo de ordenadores comparten las cuentas de usuario, y en un grupo de trabajo, cada ordenador del grupo es propietario de sus propias cuentas.
El grupo de trabajo sólo sirve como agrupador de servicios al examinar la red.
71.-¿Qué tipos de servidores existen en Windows NT?
- Controlador primario: Tiene la posesión de la base de datos de los servicios de directorio.
- Controlador de reserva: Almacena una copia de seguridad de la base de datos de los servicios de directorio. Cuando el controlador primario está fuera de servicio, el de reserva puede tomar sus funciones.
- Servidor no controlador: Servidor que pertenece al dominio pero que actúa como estación normal.
