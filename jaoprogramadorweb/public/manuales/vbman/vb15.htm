Introducción

La versión de Visual Basic 5.0 imperaba las tecnologías DAO y RDO, que siguen estando disponibles en Visual Basic 6.0. DAO o Data Acces Object dispone de un control , el DATA que nos permite el acceso a bases de datos implementadas generalmente en el motor Microsoft Jet tales como el acces o a otro tipo de bases de datos como Dbase o Paradox.

Si en vez de bases de datos locales trabajamos con bases de datos remotas existen una alternativa a DAO que es el RDO o Remote Data Object. RDO nos va a permitir el acceso a la información sin tener que usar el motor Microsoft Jet. El visual dispone de controles propios similares al DATA para trabajar con RDO.

En ambos casos, bases locales, bases remotas, DAO trabaja con una interface o controlador ODBC.

En un intento por parte de Microsoft de "universalizar" el acceso a bases de datos nace el concepto de UDA o Universal Data Access.

Esta "universalización" trae conceptos nuevos que aparecen en Visual 6.0 tales como ADO, OLE DB y RDS que en un conjunto forman el MDAC o Microsoft Data Acces Components.

Data Control

Es el control del Visual que me permite trabajar con bases de datos. Para ello es necesario arrastrar el control Data de la barra de herramientas y relacionarlo con cajas de datos que mostraran los registros de la base de datos. Para ello utilizaremos las propiedades del Data Control: DataBaseName, donde especificaremos el nombre de la base de datos, RecordSource especificaremos las tablas que usaremos de la base de datos y en Connect con la que especificaremos el tipo de base de datos(access, dBase III...).
También existen otras propiedades del data como Exclusive (para permitir el acceso a la base de datos o no), ReadOnly (no podermos modificar la base de datos dependiendo de su valor) y EofAction (que me indicará que hacer cuando lleguemos al final de la base de datos) .

Otras propiedades de controles de tipo texto nos permiten en lazar con el control data y visualizar, borrar o actualizar los datos de la base de datos como son: DataSource (para especificar el nombre del control data que enlaza con la base de datos) y el DataField (Nombre del campo de la base de datos).

Control AdoDc

En la versión 6 de Visual Basic la forma más sencilla de trabajar con una base de datos es el control AdoDc. Para tenerlo disponible en la barra de herramientas vamos al menú--> Proyecto --> Componentes y selecciono la casilla --> Microsoft ADO Data Control 6.0 (OLDB).
Para conectar el ADO con la base de datos utilizamos la propiedad ConnectionString, seleccionaremos la opción "una cadena de conexión" y pulsaremos "Generar", seleccionaremos un Proveedor (Microsoft Jet 3.51 OLE DB Provider), pulsamos siguiente, seleccionamos la base de datos y probamos si la conexión es satisfactoria. El siguiente paso es seleccionar la tabla de la base de datos mediante la propiedad RecordSource que podrá contener una tabla, una consulta almacenada en la base de datos y una sentencia SQL.
