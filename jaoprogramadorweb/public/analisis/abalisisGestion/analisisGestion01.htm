Introducción a D.A.I.

Sistemas de información.

Un sistema de información es un conjunto de elementos que están relacionados entre si y que cooperan para la consecuencia de un determinado fin.

Ejemplos de sistemas: sistema de lavado, sistema de frenado, etc.

Todo lo que afecta o rodea a un sistema se conoce con el nombre de entorno.
Un sistema puede formar parte de otros sistemas entonces se le conoce con el nombre de subsistema. Por ejemplo, el sistema lavado puede estar formado por los subsistemas lavadores y agentes de lavado. A su vez el subsistema lavadora puede estar formado por el sistema agua, el sistema electricidad, el sistema programas de lavado, etc. El subsistema agentes de lavado a su vez puede estar formado por el sistema suavizante, sistema lejía, etc.

Conceptos, objetivos, componentes y tipos:
Un sistema de información es un subsistema dentro de la empresa que permite el uso y la transferencia de información de unos subsistemas a otros.

Los elementos que componen sistemas de información son los siguientes:
- Recursos físicos: Documentos, mobiliario, archivadores, equipos de telecomunicación…
- Recursos humanos: Personal del departamento
- Reglas: Protocolos, normas, métodos, procedimientos, etc.

Si parte o toda la información y gestión de un sistema de información se realiza a través de ordenadores, el sistema de información recibe el nombre de sistema informático (SI.)

Los niveles en la implantación de un sistema de información son los siguientes:
- Transaccional: son las operaciones o transacciones cotidianas que se realizan dentro de la empresa siguiendo una clase de protocolos, establecidos por la empresa.
- Implantación: Son las relaciones que se va a establecer a las distintas operaciones de que dispone la empresa. Por Ej., si estamos en un sistema informático, será la implantación de una base de datos. Esa base de datos estará formada por tablas y esas tablas se tendrán que relacionar.
- Desarrollo: Al desarrollar una base de datos, da origen a los sistemas de información de gestión, que se conocen con el nombre de MIS (Management Information System) Con estos MIS, se pueden conseguir informes, por ejemplo, los totales de ventas de productos, vendedores, agrupándolos por los conceptos que queramos.
A partir de los MIS, se pueden tomar decisiones, y estamos ante un SSD (Sistema de Soporte de Decisiones). La utilización de bases de datos es un sistema informático, nos permite reducir la información. Al reducir la información, se minimiza la redundancia. Al no tener información repetida, se minimizan los errores tanto en alta como en escritura y actualización, aumentando la productividad, haciendo que todo el personal maneje el mismo tipo de información.

Sistemas de informaçaticos

Los ordenadores

Un sistema informático es un conjunto de elementos que permiten procesar información por medio de ordenadores. La finalidad es obtener una nueva información a partir de una ya existente.

Información de entrada -> Proceso -> Información de salida.

Los elementos de un sistema informático son los siguientes:
1- Recursos humanos: Los encargados de desarrollar, mantener, proveer de datos, y utilizar el sistema.
2- Los datos: Normalmente organizados en bases de datos.
3- Documentación: Relativa al sistema, en cuanto a su utilización.
4- Protocolos: Es decir, conjunto de procedimientos para el uso correcto de los elementos que componen el sistema, tanto de hardware como de software.

Elementos y funciones de un ordenador

Los elementos que compone un ordenador son los siguientes:

Esquema de la CPU.

Los elementos fundamentales de un ordenador son los siguientes:

Unidades de entrada: Estos dispositivos son los encargados de la entrada de información (números, textos, gráficos, dibujos, etc.). Transforman la información en señales eléctricas que son comprensibles por el ordenador.

- Tipos: Teclado alfanumérico, teclado numérico, lápiz óptico, tabletas digitalizadoras, lectoras de códigos de barras, ratones, apuntadores, escáner, cámaras digitales, dispositivos de captura de sonidos… etc.

Unidades de salida: Son los encargados de mostrar los datos de salida al usuario. Se produce el proceso inverso a los dispositivos de entrada. En este caso, las señales eléctricas se transforman en objetos comprensibles por los usuarios.
- Tipos: Monitores, impresoras, altavoces… etc.

Unidades de entrada-salida: Son dispositivos capaces de actuar tanto como entradas como salidas.
- Tipos: Pantallas, módems, tarjetas de red, unidades de almacenamiento, etc

 

Dispositivos de almacenamiento: también se les conocen como unidades de almacenamiento o memorias masivas. Se encargan de guardar los datos de los programas y todo tipo de información para ser usado por el ordenador. Para poder guardar la información en estos dispositivos, antes ha tenido que ser codificada, ya que en todos estos dispositivos, bien sean electrónicos o magnéticos, la información se tiene que representar con dos estados diferentes y opuestos entre sí (0 o 1).

Código binario
Para que los ordenadores puedan interpretar la información, esta tiene que estar codificada. Todos los códigos de este tipo tienen una característica común: Que solo pueden utilizar dos señales. Estas dos señales se representan con los dígitos 0 y 1.La razón de la utilización de este código es que todos los dispositivos del ordenador trabajan con estos dos estados, que los podemos interpretar como “activado/desactivado”, “abierto/cerrado”, “pasa/no pasa corriente”. El sistema de codificación binario se basa en el sistema de numeración binario. Este sistema utiliza dos dígitos, que son el 0 y el 1. La base de este sistema de numeración es el 2, y todos los números se forman a través de las distintas potencias de 2. La posición del digito dentro del número nos indica la potencia a la que debe ir elevada la base.

Dispositivos de almacenamiento

Ejemplos de dispositivos de almacenamiento: Cintas, discos duros, discos flexibles. A todo el conjunto de dispositivos de entrada, salida, entrada-salida y almacenamiento se les da el nombre de periféricos.

Ejercicio: Indica la lista de todos los periféricos que están conectados al ordenador que estas utilizando.
-Monitor, teclado, ratón, tarjeta de red, disco duro, lector de CD-ROM, tarjeta grafica ,
La memoria: La memoria es la responsable del funcionamiento de los programas ya que todos los programas que se van a ejecutar previamente tienen que estar cargados en la memoria. Existen dos tipos de memoria la ROM y la RAM.
Memoria ROM: Memoria de solo lectura. La información guardada en este tipo de memorias nos permite poner en funcionamiento el ordenador, son memorias no volátiles el hecho de desconectar el ordenador no afecta a su contenido.
Memoria RAM: la información se almacena en ellas temporalmente son memorias volátiles al desconectar el ordenador, desaparece toda la información que esta guardada en ellas.
Unidad central de proceso: Se compone de dos bloques fundamentales la unidad de control, y la unidad aritmético-lógica.
La unidad de control: Detecta el estado de cada uno de los elementos del ordenador. Es la encargada de recibir desde la memoria las instrucciones que se han de realizar realizando posteriormente los procesos que le son requeridos. Para sincronizar las operaciones la unidad de control posee un reloj que genera pulsos de frecuencia y se mide en megahercios.
Unidad aritmético lógica: Es la encargada de realizar todas las operaciones dentro del ordenador tanto las operaciones aritméticas, sumas, restas… como las operaciones lógicas, comparaciones, distinto, igual a, mayor o menor. Y las operaciones booleanas and or not. La unidad aritmético lógica trabaja con unas pequeñas memorias llamadas registros donde se almacenan las instrucciones que se están ejecutando estos registros se emiten en palabras en un ordenador personal estas palabras pueden contener 8, 16 o 32 bits.
Palabra: es el número máximo de bits de información que se puede transferir entre los distintos elementos que compone el ordenador.


Unidades de entrada:
-teclado: esta formado por teclas como la maquina de escribir. El teclado lo componen tres tipos de letras diferentes el alfanumérico, el de control y el numérico. En la parte central del teclado se encuentran dispuestas 6 filas de teclas, la fila superior esta formada por las teclas de función que van desde la f1 a la f12. En el resto de las teclas se encuentra el teclado alfanumérico que es como el de una maquina de escribir. El teclado de control esta formado por teclas que nos permiten desplazar el cursor por la pantalla y también nos permiten suprimir y añadir caracteres. El teclado numérico su funcionamiento es como el de una calculadora y con el podemos realizar las operaciones aritméticas básicas también existe una serie de teclas especiales, como la tecla enter, que nos permite aceptar las ordenes la tecla de control y la tecla Alt. Se utilizan para funciones específicas dentro de cada programa. Ambas siempre trabajan asociadas a otras teclas. Ctrl+Alt+supr= reset.

Ratones y dispositivos apuntadores. El ratón mas el software adecuado hace que los movimientos del ratón sobre una superficie horizontal se transforme en movimientos del puntero en la pantalla del ordenador.

Dispositivos de almacenamiento:
Como al desconectar el ordenador se pierde la información de las memorias RAM, para conservar estas información se utilizan los dispositivos de almacenamiento. Estos dispositivos pueden utilizar la tecnología magnética, la óptica y la combinación de las dos, la magnetoóptica.
Dispositivos Magnéticos: En este tipo de dispositivos la información se graba por la polarización de un material magnético. La distribución de los discos magnéticos es la siguiente.

Discos flexibles de 3 1/2: están formados por una funda protectora de plástico con una abertura en uno de sus lados, o ambas caras. La abertura esta protegida por una pieza metálica deslizante, que deja al descubierto el disco. En la cara inferior posee un elemento circular metálico, con dos muescas: una para insertarse en el eje de giro, y la otra como referencia de inicio de pista. Su nombre hace referencia a su tamaño y pulgadas, y tiene una capacidad de 1,44 MBS.
Discos duros: También se llaman fijos, ya que suelen estar dentro de la carcasa del ordenado. El número de discos ensartados determina la capacidad del disco duro. Pueden ser de 2, 3, 6, 20, 40… GBs.

Dispositivos ópticos: En lugar de utilizar la tecnología magnética, utilizan la tecnología óptica. Un lector de discos ópticos o CD-ROM tiene un funcionamiento similar a un CD de Audio. Hay dos tipos de dispositivos: de lectura-escritura, y de lectura, que son los más comunes.

Dispositivos Magneto-ópticos: Utilizan una tecnología mixta. Su gran ventaja es que pueden almacenar gran cantidad de información mediante la tecnología óptica, pero, además, los datos pueden ser modificados y borrados mediante la tecnología magnética.

BUSES

Un BUS es un canal de información formado de una serie de hilos o cables que conectan los distintos elementos de un ordenador. Los elementos de la unidad central de proceso (CPU) se conectan a través del BUS del sistema, que a su vez esta formado por el BUS de datos, el BUS de direcciones, y el BUS de control.
BUS de datos: Transporta los datos entre los distintos componentes del sistema y la unidad central de proceso.
BUS de direcciones: Transporta las direcciones de memoria y de periféricos por donde se dividen los datos.
BUS de control: Transporta señales de control y señales del estado de los periféricos. La transferencia de información puede realizarse de dos formas:
1.- En serie: La transferencia se realiza BIT a BIT,
2.- En paralelo: Dependiendo de la arquitectura del ordenador, la transferencia puede realizarse en bloques de 8, 16, 32… BITS.

Evolución de los ordenadores:
1ª GENERACIÓN DE ORDENADORES: Año 1952. El elemento básico de los ordenadores de esta generación son las válvulas de vacío. El lenguaje que utilizan estos ordenadores es el lenguaje maquina y El uso que se da a estos ordenadores es científico y militar. La información se almacena en fichas ordenadas.
2ª GENERACIÓN: 1964. El elemento básico de los ordenadores de esta generación es el transistor. Los lenguajes que se utilizan en estos ordenadores son el Cobol, el Fortran y el Ensamblador. El uso que se da a estos ordenadores es de gestión y científicos. El almacenamiento de la información se realiza en cintas magnéticas y tambores.
3ª GENERACION: 1971. El elemento básico de los ordenadores de esta generación es el circuito integrado.
4ª GENERACIÓN: 1975. Se caracteriza por la integración de la unidad central de proceso (CPU) en un microprocesador. Los lenguajes que utiliza son: Ada, Basic, Pascal, PL/1… En esta generación surgen las redes de comunicación.
5ª GENERACIÓN: A partir de 1978. En esta generación se emplea la tecnología de alta escala de integración, desarrollándose las generaciones de PCs.

SOFTWARE
Se define software de un ordenador como el conjunto de programas que es capaz de interpretar y ejecutar correctamente el ordenador. Existen tres tipos de software: Software de Control, de Tratamiento, y de Diagnóstico y Protección

1º Software de Control: Está formado por un conjunto de programas que nos permiten gestionar el ordenador. Por ejemplo, el panel de control de Windows.
2º Software de Tratamiento: Este software hace referencia al conjunto de programas que son utilizados por los usuarios. Existen dos tipos: El software de aplicación y el software de programación. Dentro del software de aplicación están todos los programas y las utilidades instaladas dentro del ordenador, por ejemplo, gestión de SO, desarrollo de documentación y proyectos, diseño visual de formularios e informes, herramientas Case….
El software de programación lo constituyen los programas que permiten el tratamiento de la información, el diseño grafico, las comunicaciones, e incluso los juegos.
3º Software de Diagnóstico y Protección: Son utilidades que están dentro del Sistema Operativo, y permiten detectar los errores. Uno de los fabricantes que vende este tipo de software es Norton. Software de Protección: este tipo de software permite detectar y eliminar los virus informáticos, tales como gusanos, troyanos, etc.

SISTEMAS OPERATIVOS
Un sistema operativo es un conjunto de programas suministrados por el fabricante del ordenador, que permiten el funcionamiento adecuado, tanto del hardware como del software de un ordenador. Existen varios tipos de Sistemas operativos:
1º Monousuarios: Los recursos del ordenador solo se utilizan por un usuario, como por ejemplo, MS-DOS, Windows, OS/2.
2º Multiusuario: Varios usuarios pueden estar conectados a un ordenador. Por ejemplo, Novell, Unix, Netware, OS/400.
3º Multitarea: Este tipo de sistemas operativos permiten la ejecución de varias tareas, de forma simultánea. Ejemplos: Windows 95, Windows 98, OS/2.
3º Monotarea: Cuando no permiten ejecutar varias tareas a la vez. Por ejemplo, MS-DOS.

Elementos de un SISTEMA operativo:

- Cargador inicial: Está formado por un conjunto de instrucciones que permiten el paso de los programas del sistema operativo desde el disco a la Memoria RAM.
- Núcleo KERNEL: Es la parte del sistema operativo que se encarga de asignar a la CPU del control de las interrupciones y de la concatenación de los procesos. Una interrupción consiste en la detección de la ejecución de un programa por la CPU, para ejecutar otro, y así sucesivamente hasta volverse a ejecutar el primer programa detenido. [Dib. Kernel] Si se solicitan varias interrupciones a la vez, la CPU indicará cuál es la prioritaria.
- El administrador de la memoria se encarga de la compactación, la segmentación, y la paginación de la memoria.
- Administradores de dispositivos de entrada/salida y de archivos: Se encarga de gestionar y mantener los periféricos y los directorios y archivos.
- Intérprete de órdenes: Ejecuta las instrucciones correspondientes a las órdenes solicitadas por los usuarios.
- Planificador de trabajos: Asigna tiempo de la CPU a cada uno de los programas que están cargados en la memoria.







MS-DOS (Sistema Operativo en Disco MicroSoft)
Es un sistema operativo monousuario y monotarea. Lo componen un conjunto de programas que se ejecutan mediante la introducción de órdenes escritas o almacenadas en un fichero. Las ordenes que se ejecutan nos dan acceso a los distintos dispositivos y a gestionar los recursos del sistema. La interfaz de MS-DOS es en modo texto, es decir, todas las instrucciones irán a través del teclado. Para poder ejecutar una instrucción la escribiremos detrás del indicador del sistema o Prompt, y luego pulsaremos Enter para ejecutarla. Las ordenes o instrucciones dentro del MS-DOS se pueden clasificar en internas o externas. Las ordenes internas están disponibles desde el inicio del sistema, es decir, se cargan en la memoria al encenderse el ordenador. Las ordenes externas se cargan en la memoria en el momento en que se van a ejecutar.
Ejemplos de ordenes de MS-DOS:
- Si queremos cambiar de unidad dentro del sistema, no tendremos más que poner el nombre de la unidad y pulsar la tecla enter.
- Si queremos saber todos los ficheros que están almacenados en el directorio actual, utilizaremos la orden DIR.
- Si queremos solicitar la hora o cambiarla, utilizaremos la orden TIME.
- Si queremos preparar un disco para poder almacenar información dentro de él, utilizaremos la orden FORMAT. Las unidades de disco formateadas se dividen en cuatro áreas: en la primera área se encuentra el sector de arranque, con los ficheros IO-SYS y MSDOS-SYS. En la segunda área esta la tabla de asignación de ficheros (FAT – File Allocation Table). En esta área se nos indicará el estado de cada uno de los sectores asociados, si esta libre, ocupado o defectuoso. En la tercera área estará el directorio raíz, y en la cuarta área estará el espacio reservado para el almacenamiento de ficheros.

LENGUAJES DE PROGRAMACION:

Definición de programa: un programa es un conjunto de instrucciones escritas en un determinado orden, que le indican al ordenador lo que tiene que realizar. Los elementos de un programa son dos:
1. Las ordenes, tareas que tiene que ejecutar el ordenador.
2. Las declaraciones, informaciones que indican los aspectos de ese programa.
Lenguaje de programación es la utilización de una sintaxis para expresar las órdenes y las declaraciones. El esquema que se sigue es el siguiente:

Lenguajes de Alto Nivel

No dependen del ordenador donde se van a ejecutar, son cada vez más parecidos al lenguaje que empleamos para comunicarnos normalmente. Lo componen dos elementos: Las órdenes, que ejecutan instrucciones y operadores, que permiten relacionar datos numéricos, textos, fechas, y elementos lógicos, y las declaraciones, que nos definen cómo son los datos (constantes, variables, o comentarios). Algunos lenguajes de alto nivel permiten la creación de bibliotecas, que contienen extensiones del lenguaje con nuevas instrucciones. Las bibliotecas pueden ser adquiridas o creadas. Los programas escritos en lenguaje de alto nivel necesitan traductores para poder ser ejecutados.

Traductores
Hay dos tipos de traductores, los compiladores y los intérpretes.
- Compiladores: Traducen los programas fuente a objeto o compilados. Los programas objeto están en lenguaje Ensamblador, y necesitan un traductor del Ensamblador para poder ser traducidos al lenguaje máquina.
- Intérpretes: Hacen que los programas sean traducidos e interpretados directamente por el ordenador. Por ejemplo, Visual Basic, no genera un lenguaje intermedio, las instrucciones son ejecutadas desde la propia aplicación. Se selecciona directamente un objeto y se ejecuta.
- Ensambladores: A cada instrucción de código fuente le corresponde una instrucción en código máquina. Para poder ser ejecutado un programa en lenguaje ensamblador, necesita ser traducido.

Lenguaje Máquina
Este lenguaje depende de la arquitectura del ordenador. Por ello, la velocidad de ejecución es muy alta. La codificación de los programas en lenguaje maquina se hace en binario, octal o hexadecimal, por lo que no es fácilmente interpretable por el programador. Este tipo de lenguajes utiliza direcciones fijas de la memoria para almacenar los datos. Un programa escrito en lenguaje máquina, para un ordenador con un procesador 286 puede ser interpretado por un ordenador con un 486, ya que ambos mantienen la misma arquitectura, pero un programa escrito en lenguaje máquina para Digital no lo interpreta un ordenador de IBM, ya que utilizan distinta arquitectura.

Generaciones en los lenguajes de programación
- Primera generación: Aparecen los lenguajes máquinas.
- Segunda generación: Aparecen los lenguajes ensambladores.
- Tercera Generación: Aparecen los lenguajes de alto nivel (Fortran, Cobol, Algol, Lisp, Pascal, “C”…).
- Cuarta Generación: Sistemas de gestión de base de datos (DB2, Oracle, SYBASE, SQL Server, Access, Fox-Pro, DBase, Visual-Basic…)


MODELOS DE DESARROLLO DE INGENIERIA DE SOFTWARE

La ingeniería del software permite la aplicación de los principios de la ingeniería al diseño de aplicaciones. Una aplicación es un conjunto de programas y otros elementos software que intentan resolver problemas. [Dib. Técnicas]

Elementos Básicos de la ingeniería.
Los elementos básicos de la ingeniería son tres: Métodos, herramientas y procedimientos.
- Métodos: Dentro de los métodos se estudia como se diseña el software.
- Herramientas: Son elementos informáticos que permiten automatizar las tareas en cada una de las fases del diseño del software. Por ejemplo, herramientas Case (Computer Aided Software Engineering | Ingeniería del software asistida por ordenador)
- Procedimientos: Es el protocolo definido durante el diseño del software. Un protocolo es un conjunto de reglas, normas, especificaciones, plazos, y controles de calidad.

Etapas o generaciones en la ingeniería del software
- Primera etapa: Desarrollo convencional. Esta dedicada a la programación. Cada programador utiliza su propia metodología. Si se tenían que realizar cambios, A veces estos cambios eran imposibles, y había que cambiar la aplicación cuando no estaba el personal que la había construido. Las fases de diseño, codificación, implantación, mantenimiento y pruebas se confundían unas con otras.
- Segunda etapa: Desarrollo estructurado. En esta etapa se diseñan los programas siguiendo una metodología de programación (las de Jackson o las de Warnier). Posteriormente aparecen metodologías orientadas al ciclo de vida de las aplicaciones, intentan disminuir en tiempo dedicado a programación y aumentar el tiempo dedicado al análisis. En esta etapa, hay dos profesionales perfectamente definidos: Analistas y Programadores. Las metodologías empleadas son las de Myers, Joudan, Constantin o Page-Jones. En esta etapa aparecen las siguientes técnicas:
o Modelado de Datos
o Diagramas de transición de estados o estudio de eventos
o Modelado del ciclo de vida de las aplicaciones
o Modelado de la construcción de prototipos
o Utilización del modelo en espiral
- Tercera etapa: Desarrollo orientado a objetos. Los elementos del mundo real se presentan como objetos sobre los que se realizan determinadas acciones.
- Cuarta etapa: Sistemas que desarrollan el ciclo de vida del software. La integración de las herramientas case permite la generación de los siguientes elementos.
o Documentación automática, indicando cual es el código fuente, y cómo se realiza la transferencia de información de unas bases a otras durante el desarrollo del software.
o Intercomunicaciones gráficas
o Mantenimiento del control de calidad sobre el producto que se está desarrollando. Algunos modelos: ISO 12207-1, El modelo Remolino, El modelo Pinball, o El modelo Métrica.

3.3 FASES DE LA INGENIERIA DEL SOFTWARE
Se definen tres fases:
- Definición del problema a resolver: Lo primero que se realiza es definir la información, indicando las caracteristicas, las restricciones, la funcionalidad. Lo segundo es definir el sistema, tanto el software como los recursos, los tiempos y los costes. Si es necesario, se crea un prototipo del sistema.
- Desarrollo de la solucion: Primero se diseñan los datos, tanto a nivel fisico como logico. Segundo: se realizara una especificación detallada del diseño que nos permita obtener los siguientes elementos:
o Código Fuente
o Las relaciones
o Los algoritmos
o Los procedimientos
Tercer: Se diseña un plan y se realizan pruebas. Estas pruebas serán de unidad, de validación y de integración, y nos permitirán detectar si se producen errores en el funcionamiento.
Cuarto: Al final de esta fase, se entrega el sistema, junto con la documentación.
- Mantenimiento de la solucion: Durante el mantenimiento del sistema, se detectaran los errores que se producen, y se realizarán las mejoras oportunas, dependiendo del funcionamiento del mismo.

- Modelo de prototipos: La construccion de prototipos se utiliza cuando se tienen los objetivos generales del sistema, pero faltan de definir todos los requisitos. Entonces se realiza un analisis del sistema, que se ira modificando según vayan apareciendo los requisitos que faltan.

4. METODOLOGÍA MÉTRICA
4.1Introducción
Métrica es una metodología de desarrollo de sistemas de información. Fue diseñada por el ministerio de Administraciones Publicas con la colaboración de otros asesores, promovida por el consejo superior de informatica para su uso en proyectos informaticos de las administraciones publicas. Por su carácter público, se permite el empleo de esta metodología a todo tipo de entidades, bien públicas o privadas, sin ninguna autorización. Como la administración pública maneja muchos equipos, que son diferentes, esta metodología es muy flexible.
4.2 Objetivos de la Metodología Metrica
La metodología metrica nos permite crear un entorno de desarrollo de sistemas que cumplen las siguientes características:
- Solucionan las necesidades y objetivos considerados como prioritarios en la definición de requisitos dados por el cliente
- Tiene en cuenta las necesidades, los presupuestos, y la duracion.
- Tiene un bajo coste de mantenimiento, ya que genera una gran cantidad de documentación.
- Utiliza metodos y herramientas en las que prima la calidad de desarrollo.
- Se adapta a la evolucion del sistema a lo largo del tiempo.

4.3 Estructura de la metodología métrica.
Las fases se dividen en módulos, los modulos, en actividades, las actividades, en tareas, y con el conjunto de todos estos elementos se obtiene un producto que es la solucion al problema.
Fases de métrica:

La metodología metrica no encaja en ninguno de los cinco modelos que hemos visto. Toma parte de ellos, y utiliza herramientas CASE y lenguajes de 4ª generación.
- Fase 0 (Plan de Sistemas de información): Se recoge la información que es necesaria. Para ello, se necesita la participación del cliente.
- Fase 1 (Analisis del Sistema): Esta fase se divide en dos modulos, el modulo 1 (Analisis de requisitos del sistema, es decir, que restricciones nos impone el sistema) y modulo 2 (especificación funcional del sistema). Cuando se termina este modulo (segundo), se obtiene un documento con la siguiente información: Definición de subsistemas, definición de los datos del sistema, interfaces de usuario, prototipos y especificación de entrega.
- Fase 2 (Diseño del sistema): En esta fase se encuentran las especificaciones fisicas, es decir, el entorno tecnologico donde se va a implantar el sistema.
- Fase 3 (Construccion del sistema): Con la ayuda de las fases anteriores, se conseguira el funcionamiento correcto de los diversos programas, ya desarrollados.]

- Fase 4 (Implantación del sistema): Estara formado por el conjunto de pruebas a las que se somete al sistema para obtener su correcto funcionamiento. Se somete a pruebas tanto a los elementos logicos como a los elementos físicos. Si el resultado es satisfactorio, se implanta el sistema.
