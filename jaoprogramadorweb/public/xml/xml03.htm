¿Qué es el XML?

Es el estándar de Extensible Markup Lenguaje. XML no es más que un conjunto de reglas para definir etiquetas semánticas que nos organizan un documento en diferentes partes. XML es un metalenguaje que define la sintaxis utilizada para definir otros lenguajes de etiquetas estructurados.

De hecho, HTML es simplemente un lenguaje, mientras que XML como se ha dicho es un metalenguaje, esto es, un lenguaje para definir lenguajes. Y esa es la diferencia fundamental, de la que deriban las demás, que iremos viendo a lo largo del documento.

 

XML proporciona a las empresas el camino para el intercambio de información. Por eso estas prefieren los formatos XML a otros porque es mas fácil de interpretar y de manipular se podíra decir que es casi auto descriptivo. De esta forma, si se recibe una cadena de texto normal sin ningun identificador y la tenemos que interpretar resulta difícil saber que es exactamente , pero si ese texto va acompañado de unas marcas, si que podemos identificarla(transforma los datos a información descriptiva). Es un meta markup language(metalenguaje de marcas) y es extensible ya que no es un formato prefijado como HTML.

XML refuerza el impacto de Internet en el mundo empresarial ya que se integran de esta forma más rápidamente. Es decir, si queremos modificar la prsentación o contenido de páginas se requiere menos tiempo y recursos humanos utilizando XML que otro formato.

 

Es el futuro lenguaje de la web aunque por ahora no lo sustituirá. Esto es debido a que los navegadores actuales solo interpretan HTML. Existen browsers que dicen entender XML , aunque en realidad tienen que transformar este lenguaje para poder presentarlo.

 

El uso de XML elimina la libertad del HTML que es anárquico, y nos presenta el XHTML que debe seguir unas normas. El Extensible Markup Lenguaje es un estándar desarrollado por el grupo W3C o Word Wide Web Consortium (www.w3.org) para textos estructurados.. Este grupo se dedica a definir estandares en los que las empresas de software se basan posteriormente para crear sus aplicaciones.Existen otros subgrupos del W3C como es el grupo OASIS que pretende la estandarización de los documentos y así permite el intercambio de estos facilmente y las complicaciones que se producirían con documentos con diferentes estructuras. Este grupo tiene ya definido un catálogo de XML con DTD estandarizados, de esta forma las ramas sanitarias (HL7), químicas farmaceúticas (CML) financieras(VisaXML, IFX, BIPS), del turismo/viajes (OTA), de la prensa/publicitarias (aXML), etc… puede intercambiarse documentos con un DTD ya definido y estandarizado, y por lo tanto sin complicaciones.

 

Actualmente hay 2 versiones de XML:1.0 y 1.1 aunque la mayoría de los parsers (procesadores) por ahora sólo reconocen la versión 1.0.

 

El XML separa contenido de estructura, es decir podemos saber si un documento está bien formato sin saber qué es lo que contiene. Todo elemento del documento tiene un principio y un final, y entre esas dos etiquetas tendremos en valor del elemento, aunque tambien puede contener otros elementos.

 

<?xml version=” 1.0” encoding=”ISO-8859- 1” ?>

<informe>

<fecha>17/5/1990</fecha>

<hora>14:00</hora>

<area>

<ciudad>León</ciudad>

<region>Castilla Leon</region>

<pais>España</pais>

</area>

<mediciones>

<cielo>claro</cielo>

<temperatura escala=”C”>25</temperatura>

</mediciones>

 

</informe>

 

En el ejemplo <area> es un elemento que contiene otros elementos y <ciudad> es un elemento que contiene un valor.
