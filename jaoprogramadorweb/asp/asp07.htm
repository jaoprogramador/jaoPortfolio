Funciones y subrutinas en ASP

Funciones y subrutinas

Cuando escribo un código que se volvera a repetir puedo utilizar subrutinas o funciones.
Se pueden pasar parámetros a dichas funciones . La subrutina no devuelve datos, la función si.


Sub nombre_subrutina (parametros,…)
comandos
End sub

Para llamar a la subrutina hay dos maneras

Call nombre(valor_parametros, )

Nombre valor_parametros,…


Para salir de la subrutina sin ejecutar todo el codigo exit sub (puede haber varios)
Las variables declaradas en la subrutina son publicas por defecto, pueden modificarse desde otra subrutina, por lo que conviene hacerlas privadas.

Function nombre_funcion (parametros,…)
Comandos
Nombre_funcion=valor_devuelto
End function

Para llamar a la funcion asignamos el valor a otra variable o lo utilizamos en otra función

Var=nombre_funcion(valor parámetros,…)
Response.write(nombre_funcion(valor parametros,…))

Cuando hay un error en la funcion, si debe devolver un valor numerico devuelve 0, si devuelve un texto devuelve “”, si devuelve una referencia a un objeto devuelve NOTHING.


Para salir de la funcion sin ejecutar todo el codigo exit function (puede haber varios)
Las variables declaradas en la subrutina son publicas por defecto, pueden modificarse desde otra subrutina, por lo que conviene hacerlas privadas.

Ejemplo

<%
function areac(l)
   areac=l*l
end function
dim lado
lado=5
response.Write("El area del cuadrado de lado "&lado&" es "&areac(lado))
%>

Las rutinas y funciones se suelen agrupar en librerias. Las librerias unicamente pueden contener codigo Asp, se crean dentro de un archivo *.asp . con subrutinas y funciones.
Para incluir una librería en asp

<!--#include file=”area.asp”-->

Crearemos una página que llame a una rutina que calcule el area:

<%
function arear(a,b)
